Where Agentic Stacks Fits

The landscape has protocol layers and tool registries, but nobody owns the "composed capability pack" layer

Current Landscape

Protocol Layer

How agents talk to tools

MCP — tool discovery & invocation
A2A — agent-to-agent delegation
OpenAPI — API schema description
Status: Maturing, but fragmented

Single-Tool Registries

Individual tools & servers

Smithery — MCP server directory
mcp.run — WASM-sandboxed MCP
Composio — hosted integrations
GPT Store — closed ecosystem
Status: Early, fragmented, no versioning
THE GAP

Composed Capability Packs

Domain expertise bundles

No versioned skill packs
No composable knowledge bundles
No agent-queryable registry
No trust/signing model
This is where Agentic Stacks lives
Proven Patterns to Borrow

From Helm Charts

  • Dual versioning (pack version + target version)
  • Dependency declaration with SemVer ranges
  • Lock files for reproducibility
  • OCI registry distribution
  • Immutable published versions

From Ansible Galaxy

  • Namespace.collection naming
  • Two-level deps (pack-level + project-level)
  • Formal deprecation policy
  • Selective skill inclusion (not all-or-nothing)

From Kustomize

  • Base + overlay composition model
  • Patch/override without forking
  • "Same base, different targets" pattern

From HuggingGPT / Voyager

  • Agents query registries by natural language
  • Agents accumulate & compose skill libraries
  • Schema + examples = enough for discovery
